SICP-2.2笔记（除图形化语言）

发表于 2021-03-21 分类于 学习笔记 阅读次数： Changyan：

前言

早就听说SICP是本奇书，也无数次下定决心学习。可是，也许由于是个人能力所限，学习之路总是充满了坎坷。

大一上时在图书馆借了中文版。当时初读之欣喜若狂，觉其中精妙之处甚多。此书关于迭代与递归的分析绝妙，其中所讲的高阶函数，lambda演算也让只知道一点点c语言的我大开眼界。但也许因为急功近利，读之过快，前文尚未消化，眼就瞟下下文。加之当时视野较窄，不会查找资源，没有同学与伴，所以我居然连一个scheme解释器都找不到，只能在纸面上编程，因而对于语言的理解就更差了。所以一节一节下来，我未消化的知识越来越多，不懂的就越来越多，最终第一章尚未读完，我就已支撑不下去，遂作罢。

大一上的失败给我两个教训：

  1. “独学而无友，则孤陋而寡闻。”在学习时的环境与交流也时必须的。
  2. 资源查找是一项很重要的能力。

寒假之时，我痛定思痛，心有不甘。想到UCB的python版SICP现在已成系统，网上教学资源丰富，加之南大计科也引入了这门课，所以我能轻松找到这门课的大量资源，于是便听UCB的网课，看网上的课本，做UCB的homework和lab。UCB课本学习曲线较原版SICP舒缓得多，于是我进度很快，学完了过程抽象与数据抽象两章。但是我一高兴，就急功近利，想马上学完。于是我不做作业也不看视频，就猛看课本，结果和大一上一样，越看越看不懂，所以也是半途而废。另外，在学习UCB SICP得过程中，因课本中文翻译奇差，我不得不硬着头皮去看英文版。刚开始艰难，后来却越来越上头，觉得英文版比中文版清楚太多。

寒假的失败也给了一点点收获：

  1. 在学习网课时，我大量在网上查找资源，结果兴趣又被很多乱七八糟的小东西吸引。我假期尝试买了服务器，搭了梯子，建了Blog，还学了许多小工具。这些虽然看起来用处不大，却提升了我查找资源的能力，开阔了我的视野。
  2. 从此我认识到了英文的重要性。并下决心，以后凡是资料有英文原版因当优先看原版。

经过两次的失败，SICP已经几乎成为我的梦魇。但我还是下决心去克服它。我这次直接读MIT SICP英文原版。可能因为之前积攒的功力，我的进展也十分顺利。对比之后感到MIT版比UCB版高妙很多，也难许多。

可是随着章节推进，我在2.2节又陷入了困境。2.2节抽象程度极高，一个函数定义接着一个函数定义。刚才的新定义马上就被当作下一次抽象的材料。以我的记忆力和急性子，读了根本记不住，又何谈理解？于是我决定开始做一点笔记，把关键的代码保存下来，以供查阅使用。

估计在2.2节之后，我每一节都会做一点笔记。

    Sequences as Conventional Interfaces

  • the use of conventional interfaces.

  • 我们能从程序中抽象出一些普遍模式（利用高阶过程），而这种抽象依赖于我们我们如何组织数据结构。

  • 一个例子：

  • A signal-processing engineer would find it natural to conceptualize these processes in terms of signals flowing through a cascade of stages, each of which implements part of the program plan。

The first program

  • enumerates the leaves of a tree;
  • filters them, selecting the odd ones;
  • squares each of the selected ones; and
  • accumulates the results using +, starting with 0.

The second program

  • enumerates the integers from 0 to n;
  • computes the Fibonacci number for each integer;
  • filters them, selecting the even ones; and
  • accumulates the results using cons, starting with the empty list.

  • Sequence operations

  • The key to organizing programs so as to more clearly reflect the signal-flow structure is to concentrate on the “signals” that flow from one stage in the process to the next. If we represent these signals as lists, then we can use list operations to implement the processing at each of the stages.
